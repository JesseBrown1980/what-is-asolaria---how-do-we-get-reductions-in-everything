## VERBATIM

This is essentially a **red-team role** baked into the agent loop — not chaos for chaos's sake, but a forcing function that makes the consensus agents actually justify themselves.

**Implementation options in your stack:**

1. **Prompt weight** — Add a role: "contrarian" agent that receives the other agents' outputs and specifically attacks them. Cheap to add via Omnispindle as a tool.
2. **Temperature dial** — Run the same prompt at temp: 0.9-1.2 in parallel with the main agents, then score divergence. High divergence = surface to human review.
3. **Constraint injection** — Force the madness agent to solve under artificial restrictions ("you can't use a database", "no external calls") — weird constraints unlock lateral paths.
4. **Churn-weighted skepticism** — Feed the cartogomancy data in: high git churn + high complexity = the madness agent gets louder about that module.
