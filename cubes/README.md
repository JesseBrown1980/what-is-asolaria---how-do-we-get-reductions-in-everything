# Cubes

This folder is the public map of how Asolaria uses **cubes** as a materialization and reasoning form.

**Honest frame:** a cube here is **not automatically a folder** and **not automatically a live daemon**. Depending on the layer, a cube can be:

1. an **inference structure**,
2. a **descriptor / receipt**,
3. a **tuple-range address model**,
4. a **bounded live worker pattern** over a much larger logical space.

This folder corrects a recurring mistake:

> if there is no giant folder tree, that does **not** mean the system was not formalized.

The public evidence now supports three cube-formalization surfaces:

1. **Tensor-collapse / inference cubes**  
   The cube becomes an inference engine and collapses large Omni-Shannon beat tensors into a small signal set.
2. **Tuple-range / room-layer cubes**  
   Rooms are represented as **BEHCS tuple ranges** and shard/rotor topology, not only as literal folders.
3. **Bounded runtime cubes**  
   A small number of live chambers, queues, and absorbers work over a much larger address space.

## The room-sector cube question

> Where are the `ROOM-SECTOR-100K-SCALED-ROTOR` + 100 shards + 12 lanes formalized — in folders, cubes, or ranges?

**Answer:** they are formalized into the cube/codebook + tuple-range model. The folder view is only one descriptor layer.

## Three concrete ways something is "in the cubes"

### 1. Tuple-range addressing

The live fabric-revolver reports:

- `architecture.tuple_ranges_are_backend_nodes = true`
- `architecture.real_worker_slots_are_chambers = true`
- `architecture.active_chambers = 8`

So a room is a **BEHCS-1024 tuple range** and only a small bounded chamber set is live at once.

### 2. Shard-quant cube receipts

Each of the **100 shards** carries a `hyperbehcs.quantized_substrate_receipt.v1` that folds its **100 rooms** through four codecs:

- **Johnson-Lindenstrauss** — 8→4 projection
- **turbo quant** — 8-bit codes / 256 levels
- **polar quant** — radius-angle paired quantization
- **triple quant** — coarse / medium / fine spherical lanes

Each receipt is `descriptor_only`, `software_only`, and sha16-fingerprinted. Real sample:

- [sample-shard-quant-cube.annotated.json](sample-shard-quant-cube.annotated.json)

### 3. Tensor-collapse and bounded runtime carriers

The tensor-collapse line shows the cube as an **inference engine**, while the later runtime stack shows the same worldview carried by:

- revolver chambers,
- omnispindle queues,
- omniflywheel absorption,
- SUPER OS map surfaces.
