3. **what controls keep it honest**
4. **how acer and liris divide the work**

## 4. Controls that keep it honest

The papers and runtime surfaces show a real control stack, not free-form mysticism.

### Research controls

- **Boundary Is Data Rule** — absence/silence can be evidence
- **cascade revalidation** — later ground truth can revise earlier verdicts
- **self-falsifiable / self-healing doctrine** — contradiction is part of the method
- **new dimensions by reflection** — `D26`, `D30`, `D32` promoted because the run needed them

### Runtime controls

- **bounded chamber count** — revolver keeps only a small live worker pool
- **`tuple_ranges_are_backend_nodes=true`** — address space is larger than live workers
- **`range_packet_not_process_per_node`** — no process per logical node
- **omniflywheel absorption** — partial results are absorbed/classified rather than forcing fake completeness
- **`no_fake_model_output`** surfaces in revolver receipts

### Constitutional/operator controls

- operator authorization
- hard gates / hookwall discipline
- held-safe / not-live-now chamber states
- descriptor vs live-run boundary

This is why the public reading must always include both sides:

- **capable**
- **and**
- **currently live**

Those are not the same claim.
